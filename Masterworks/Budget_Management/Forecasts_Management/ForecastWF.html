Forecast Workflow Stages

The following table provides the workflow stages (current and subsequent stage) of the Forecast for Budget Estimate and Budget Estimate Revision records.

For information on setting a workflow stage to the next stage, refer to Workflow Stage Transitions.

PhaseCurrent Workflow StatusActionSubsequent Workflow StatusComments
1DraftCompleteCompleted-
2CompletedPublishPublishedThe workflow status of the associated budget estimate is set to Published.

In the BUDGET ESTIMATES list page, the Release to Planning check box is selected automatically to indicate that the project is ready to be included in a program.

The workflow status of the previously published forecast, if exists, is set to Un-Published.

Re-DraftDraft-
3PublishedUn-PublishUn-PublishedIn the Un-Published workflow status, you can update the budget estimate.
Approve-You cannot perform this workflow action as approval of a forecast is driven by the approval of the program.

The forecast and the associated budget estimate can be approved only by adding the published project to a Program and then adding the program to a 2-level Program. Approving the 2-level Program automatically approves the associated programs, and the published forecast and budget of projects part of these programs.
